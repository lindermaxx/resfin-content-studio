## Story

**Como** Max Linder,
**Quero** que o projeto Next.js seja criado com toda a configuração base,
**Para que** eu possa acessar o app pelo browser e a equipe de desenvolvimento possa trabalhar com uma base sólida.

---

## Acceptance Criteria

- [ ] AC1: Projeto Next.js 14 criado com App Router, TypeScript, Tailwind CSS e ESLint
- [ ] AC2: shadcn/ui instalado e configurado com tema neutro
- [ ] AC3: Estrutura de pastas criada: `src/app/`, `src/components/`, `src/lib/`, `src/types/`
- [ ] AC4: Layout base com sidebar ou navbar com links para: Research, Copy Studio, Review, Image Studio, Pipeline
- [ ] AC5: Página inicial (`/`) exibe boas-vindas com o nome do app e os 5 módulos como cards clicáveis
- [ ] AC6: Repositório criado no GitHub (`resfin-content-studio`) com branch `main`
- [ ] AC7: `.env.local.example` criado com todas as variáveis necessárias documentadas

---

## Scope

**IN:**
- `npx create-next-app` com flags corretas
- Instalação e configuração do shadcn/ui
- Layout base (sidebar ou top nav) com os 5 módulos
- Página inicial estática (sem chamadas de API ainda)
- Repositório GitHub
- `.env.local.example`

**OUT:**
- Nenhuma integração de API nesta story
- Nenhum conteúdo dinâmico
- Deploy no Vercel (é a Story 1.2)

## Tasks

- [ ] 1. Executar `npx create-next-app@latest resfin-content-studio --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"`
- [ ] 2. Instalar e inicializar shadcn/ui: `npx shadcn@latest init`
- [ ] 3. Instalar componentes base do shadcn/ui: `button`, `card`, `badge`, `separator`, `scroll-area`
- [ ] 4. Criar estrutura de pastas: `src/lib/`, `src/types/`, `src/components/ui/` (já criada pelo shadcn), `src/components/layout/`
- [ ] 5. Criar `src/components/layout/Sidebar.tsx` com navegação para os 5 módulos
- [ ] 6. Atualizar `src/app/layout.tsx` para incluir o Sidebar
- [ ] 7. Criar `src/app/page.tsx` com cards dos 5 módulos (estático)
- [ ] 8. Criar páginas vazias (placeholder): `/research`, `/copy`, `/review`, `/image`, `/pipeline`
