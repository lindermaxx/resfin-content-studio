# Story 1.2 — Deploy Vercel + Vercel KV + variáveis de ambiente

> **Status:** Draft

## Story

**Como** Max Linder,
**Quero** que o app esteja acessível via URL pública no Vercel com o Vercel KV conectado e todas as variáveis de ambiente configuradas,
**Para que** eu possa acessar o ResFin Content Studio pelo navegador em qualquer lugar, sem precisar rodar o projeto localmente.

---

## Acceptance Criteria

- [ ] AC1: Projeto importado no Vercel conectado ao repositório GitHub `lindermaxx/resfin-content-studio`
- [ ] AC2: Deploy automático ativo — push na branch `main` dispara novo deploy
- [ ] AC3: App abre sem erros na URL pública gerada pelo Vercel (ex: `resfin-content-studio.vercel.app`)
- [ ] AC4: Vercel KV provisionado e vinculado ao projeto no Vercel Dashboard
- [ ] AC5: Variáveis de ambiente configuradas no Vercel: `ANTHROPIC_API_KEY`, `GOOGLE_AI_API_KEY`, `GOOGLE_IMAGE_MODEL`, `GOOGLE_TEXT_MODEL`, `OPENAI_API_KEY`, `KV_URL`, `KV_REST_API_URL`, `KV_REST_API_TOKEN`, `KV_REST_API_READ_ONLY_TOKEN`
- [ ] AC6: Pacote `@vercel/kv` instalado e cliente KV criado em `src/lib/kv.ts`
- [ ] AC7: `npm run build` continua passando após adição do `@vercel/kv`

---

## Scope

**IN:**
- Importar repositório GitHub no Vercel e fazer primeiro deploy
- Provisionar Vercel KV e vincular ao projeto
- Configurar variáveis de ambiente no Vercel Dashboard
- Instalar `@vercel/kv` e criar `src/lib/kv.ts` com cliente pronto para uso

**OUT:**
- Nenhum uso real do KV nesta story (apenas setup — uso começa no EPIC-04)
- Nenhum domínio customizado (usar URL padrão Vercel)
- Nenhuma configuração de CI/CD além do deploy automático padrão do Vercel

## Dev Notes

- O cliente KV em `src/lib/kv.ts` deve importar de `@vercel/kv` e exportar a instância pronta
- Em ambiente local (sem KV), o cliente pode retornar `null` graciosamente — o KV só é exigido em produção
- As variáveis `KV_URL`, `KV_REST_API_URL`, `KV_REST_API_TOKEN` e `KV_REST_API_READ_ONLY_TOKEN` são geradas automaticamente pelo Vercel quando o KV é conectado
- `GOOGLE_IMAGE_MODEL=gemini-3-pro-image-preview` e `GOOGLE_TEXT_MODEL=gemini-2.0-flash` já estão no `.env.local.example`
- Não expor nenhuma API key no lado cliente — todas as vars sem prefixo `NEXT_PUBLIC_` ficam apenas no servidor
