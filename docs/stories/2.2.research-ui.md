## Story

**Como** Max Linder,
**Quero** uma interface visual no módulo Research para ver, selecionar e confirmar temas,
**Para que** eu possa escolher um trending topic ou digitar meu próprio tema e seguir para o Copy Studio.

---

## Acceptance Criteria

- [ ] AC1: Botão "Buscar Trends da Semana" chama `/api/research/trends` e exibe loading state
- [ ] AC2: 10 cards de trends exibidos com: título, plataforma (badge), pilar (badge) e hook
- [ ] AC3: Max pode selecionar 1 card (highlight visual) para trabalhar
- [ ] AC4: Campo de texto "Meu próprio tema" permite digitar tema manualmente (ignora os cards)
- [ ] AC5: Textarea "Colar rascunho de copy" permite colar texto para refinar
- [ ] AC6: Botão "Ir para Copy Studio" ativo apenas quando há tema selecionado OU tema manual preenchido
- [ ] AC7: Ao clicar em "Ir para Copy Studio", navega para `/copy` passando o contexto (tema + rascunho se houver)

---

## Scope

**IN:**
- Página `/research` completa substituindo o placeholder
- Componente de card de trend
- Estado local com `useState` (tema selecionado, tema manual, rascunho)
- Navegação para `/copy` com contexto via `localStorage` ou `sessionStorage`
- Loading state e mensagem de erro visual

**OUT:**
- Persistência no Supabase (só o post aprovado será salvo — EPIC-04)
- Histórico de pesquisas
- Filtro por pilar

## Dev Notes

- Usar `useState` para: `trends`, `loading`, `error`, `selectedTrend`, `manualTema`, `rascunho`
- Card selecionado: borda colorida + check icon (zinc-900 border + CheckCircle)
- Badge de plataforma: variant `outline`
- Badge de pilar: mapeado para label legível (ex: `educacao_financeira` → "Educação Financeira")
- `sessionStorage` key: `resfin_research_context` com shape `{ tema, pilar, hook, rascunho, source: 'trend' | 'manual' }`
- Página `"use client"` — faz chamada à API no browser
