## Story

**Como** sistema,  
**Quero** persistir os posts gerados com status e metadados no Supabase,  
**Para que** o Pipeline opere com dados reais e histórico.

---

## Acceptance Criteria

- [ ] AC1: `POST /api/posts` cria post com status inicial
- [ ] AC2: `GET /api/posts` retorna posts ordenados por data
- [ ] AC3: `PATCH /api/posts/:id` atualiza campos editáveis
- [ ] AC4: `PATCH /api/posts/:id/status` muda status com validação
- [ ] AC5: Alterações de status registram evento em log de atividade

---

## Tasks

- [ ] 1. Definir tipo `Post` em `src/lib`
- [ ] 2. Criar routes API de posts (create/list/update/status)
- [ ] 3. Integrar `src/lib/supabase.ts` nas routes server-side
- [ ] 4. Padronizar erros em português para UI
- [ ] 5. Cobrir casos de post inexistente e status inválido

---

## Definition of Done

- [ ] CRUD mínimo de posts funcional
- [ ] Transição de status persistida
- [ ] Log de atividade gravado
- [ ] `npm run lint` sem erros
